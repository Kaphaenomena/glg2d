JOGLJ2D OpenGL accelerated Graphics2D

JOGLJ2D is a Graphics2D implementation that uses OpenGL to implement basic Java2D drawing functionality.

The most recent versions of the Oracle JVM have an OpenGL pipeline to accelerate drawing on some platforms. But the pipeline doesn't use the full power of OpenGL since it doesn't make use of OpenGL's ability to draw primitives directly, like glRect(). The Oracle implementation processes most of the Graphics2D calls in Java then sends very primitive commands to the OpenGL system.

JOGLJ2D takes a different approach. Our object is to accelerate the most common ways that applications use the Graphics2D drawing. For example, when drawing shapes, we switch on the Stroke implementation and use a specialized class for BasicStroke. For more advanced functionality, like AlphaComposite, we use fragment shaders. In the future, even some of the line drawing could be implemented using geometry shaders.

We use the JOGL 1.1.1a libraries for making OpenGL calls. As soon as the jogamp project hits a major release, we will move to the updated JOGL libraries.

Implementation Status

Not all Graphics2D methods have been implemented and accelerated using JOGL calls. In general, we try to implement the most commonly-used functions first. If you have a request for a feature or want to report a bug, send me an email at borkhobs+joglj2d (at) gmail.
